Overview

This project is a python data analysis project conducted to learn how runners performed in the San Francisco Marathon race of 2017.

In order to have an insightful understanding of runners' performance, the San Francisco Marathon race results and Strava applicaiton users' running records were used. Those information were gathered from the websites using techniques such as data scraping, data cleansing, and data visualization through python programing.

Following questions were used to understand the sampled data demographics, significans of any outliers, and potential reasons behind how runners performed on the marathon race day.

Key questions:

  • What does the Strava user population look like compared to overall marathon runners?
  • How did the Strava users perform at the race day compared to overall marathon runners?
  • How did top 20 Strava users perform on the race day?
  • What can we do to race for better performance?

Data source

Total of 1317 user running data was captured using data scraping technique to perform data gathering from the Strava website as well as the official San Francisco Marathon result page. The inforamtion was first formatted, and then used to learn about the different demographics of the Strava users compared to the 2017 San Francisco Marathon official results.

Strava user demographics

Using the official San Francisco marathon race result, scatter plot was produced to show the runners' overall race results by gender.

From this analysis, the scatter plot is most dense between overall race result of 2:46:40 to 6:56:40.

There were no apparent observation of race performance based on gender, however there is a positive corrolation between runners' age and the overall time to complete the marathon.

We observed that the younger the runner is, the more likely the overall performance result in faster completion time.

Reference: SFmarathon2017-Final.ipynb
